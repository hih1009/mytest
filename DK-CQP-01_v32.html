설정

Google Cloud Console에서 발급받은 OAuth 2.0 클라이언트 ID를 입력하세요.

Google Cloud Console → API 및 서비스 → 사용자 인증 정보 → OAuth 클라이언트 ID(웹 애플리케이션)에서 발급받고,
이 파일을 실제로 호스팅할 주소(예: https://본인계정.github.io)를 "승인된 자바스크립트 원본"에 등록해야 합니다.
파일을 그냥 더블클릭해서 열면(file://) Google 로그인이 동작하지 않습니다. GitHub Pages 등으로 배포해서 사용하세요.
Drive API는 drive.file 범위만 사용하며, 이 앱이 만든 파일에만 접근합니다.
Google Drive에 생성될 최상위 폴더명입니다.
특정 팀/공유 폴더 안에 저장하고 싶다면 그 폴더의 링크나 ID를 붙여넣으세요. 비워두면 내 드라이브 최상위에 생성됩니다. (앱이 접근하려면 해당 폴더가 이 계정과 공유되어 있어야 합니다.)
Firebase 콘솔 → 프로젝트 설정 → 일반 → "내 앱"에서 나오는 firebaseConfig 값을 JSON 형식(따옴표 포함)으로 붙여넣으세요.
Authentication에서 Google 로그인 방법을 사용 설정하고, Firestore Database를 생성해야 합니다.
신규 가입자는 승인 전까지 로그인할 수 없으며, 관리자(hih100930@gmail.com) 계정으로 로그인하면 이 설정 화면 아래에 "가입 승인 대기" 목록이 나타나 여기서 역할을 지정하고 승인할 수 있습니다. (Firebase 콘솔 → Firestore → signers 컬렉션에서 직접 role / status 필드를 수정해도 됩니다.)
